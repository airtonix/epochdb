AboutWhy and How

Intro

So I was out there on Green Mountain one day, about to start the whole creative process of base-building. I needed some reference matieral to remind myself what was required to build all the various parts (because why memorise something when you can look it up right?).

However one this day. dayzepoch.com was down for backup.

So I said, "that's it" and started gathering as much data as I could find.

The Plan

I had this plan :

  • Host the site from github pages (becuase I love git)
  • Use AngularJS as the clientside framework, avoid Jquery
  • Because github only allows static websites, and angular doesn't have a yaml consumer yet, I'd have to store all the data as JSON. No problem.
  • Develop the application locally using Wintersmith, a static website generator that consumes folder tree of markdown/json/yaml files and lets you use what ever templating language you want (I went with Jade, because I like the time saving data structure of yaml)
  • I would use Sass and Foundation. But because I was staying as far away from Ruby as I could manage (wintersmith is a nodejs tool), I would use Bourbon instead of Compass.
  • I'd make use of Grunt, Npm and Bower because manually finding and downloading stuff is such a boring task.
  • Finally, my deployment strategy is a `git add --all && git commit -am "changelog message" && git push`

Difficulties

  • Contributing will be different. You'll need a Github account (which is free)
  • No server side logic, which is a blessing in disguise. All the critical features requiring server side tools are already provided by Github.
